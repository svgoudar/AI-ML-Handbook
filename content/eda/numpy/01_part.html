Overview#

ndarray (N-Dimensional Array)#

  • Homogeneous data type, unlike Python lists.

  • Stored in contiguous memory blocks.

  • Provides efficient indexing, slicing, broadcasting.

Shape#

  • Tuple of array dimensions.

  • Example: (3,4) means 3 rows, 4 columns.

Data type (dtype)#

  • Each element has a fixed type (e.g., int32, float64).

  • Ensures predictable memory use and fast computation.

Itemsize#

  • Number of bytes used by each element.

  • Example: int32 → 4 bytes, float64 → 8 bytes.

Strides#

  • Number of bytes to move in memory to step to the next element in each axis.

  • Controls how slicing and views work.

Contiguity (C-order vs Fortran-order)#

  • C-order: Row-major (last index changes fastest).

  • Fortran-order: Column-major (first index changes fastest).

  • Impacts performance of iteration and compatibility with external libraries.

7. Views vs Copies#

  • View: New array object points to same data (uses strides).

  • Copy: New array with duplicated memory.

  • Example: slicing → view; some reshapes → copy.

8. Broadcasting#

  • Rules to allow operations on arrays of different shapes without copying data.

  • Example: (3,1) with (1,4) → result shape (3,4).

9. Universal Functions (ufuncs)#

  • Vectorized operations that apply element-wise.

  • Example: np.add, np.exp, np.sin.

  • Avoid explicit Python loops → faster.

10. Memory Efficiency#

  • NumPy arrays are compact due to fixed dtype and contiguous memory.

  • Enables interface with low-level C/Fortran routines.
